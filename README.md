# 🎮 五子棋 (Gomoku) - 基于 Pygame 与 Minimax算法

## 📖 项目简介
这是一个使用 Python 和 Pygame 库开发的经典五子棋人机对战游戏。玩家执黑棋先手，AI 执白棋。
本项目的核心亮点在于其 **AI 博弈算法**：采用了**极大极小值算法 (Minimax)** 结合 **Alpha-Beta 剪枝**，并引入了**迭代加深**、**历史表启发**和**候选点生成**等专业优化策略。同时，利用多线程技术确保 AI 在后台深度思考时，游戏 UI 界面依然保持丝滑流畅。

## ✨ 核心特性
- **精美的图形界面**：基于 Pygame 绘制 15x15 标准棋盘，包含星位标记、带描边的棋子渲染、鼠标悬停半透明落子预览，以及最后一步落子的红点高亮提示。
- **强悍的 AI 对手**：
  - **Minimax + Alpha-Beta 剪枝**：高效的博弈树搜索，模拟双方最优策略。
  - **迭代加深搜索 (Iterative Deepening)**：在限定时间（2秒）内逐层加深搜索深度，确保总能返回当前最优解。
  - **启发式评估函数**：基于棋型（如活四、冲四、活三等）和位置权重（越靠近棋盘中心分数越高）进行精准打分。
  - **历史表 (History Heuristic)**：记录历史最优节点，优化搜索树的遍历顺序，大幅提高剪枝效率。
  - **杀招/防杀优先**：在深度搜索前，优先扫描“一步成五”或“必须防守”的绝对关键点。
- **流畅的交互体验**：采用 `threading` 多线程异步处理 AI 计算，彻底告别 AI 思考时界面卡死的问题。
- **便捷的控制**：支持鼠标点击落子，一键重启游戏。

## 🎮 操作说明
- **落子**：移动鼠标预览落子位置（会显示半透明阴影），**鼠标左键点击**棋盘交叉点即可落子。
- **重新开始**：在游戏进行中或游戏结束后，按下键盘 **`R`** 键即可清空棋盘，重新开始对局。
- **退出游戏**：点击窗口右上角的关闭按钮。

## 🧠 AI 算法原理解析
1. **候选点生成 (Candidate Generation)**：
   AI 不会盲目遍历整个 15x15 的棋盘，而是只选取已有棋子周围 `2格` 范围内的空位作为候选点，并结合位置权重进行排序，截取前 20 个最优候选点，极大减少了搜索空间。
2. **棋型评估 (Evaluation Function)**：
   通过统计各个方向的连子数以及两端是否被堵（`block` 数量），映射到预设的分数表（如 `10000000` 代表五连，`50000` 代表活三等），得出盘面总分。
3. **多线程调度**：
   玩家落子后，主线程触发 `ai_done_event`，后台线程 `ai_think` 开始计算最佳落子点。计算完成后通过线程锁安全地将结果传递给主线程进行渲染。

## 📂 代码结构简述
- **`Gomoku` 类**：封装了棋盘状态、落子逻辑、胜负判定、评估函数及核心 AI 搜索算法。
- **`draw_board` 函数**：负责 Pygame 界面的绘制、UI 文本渲染及状态提示。
- **`main` 函数**：游戏主循环，处理事件监听（鼠标、键盘）、多线程调度及帧率控制 (`clock.tick(30)`)。
